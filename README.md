# CreditWise AI

CreditWise AI is a full-stack credit risk prediction application built with a FastAPI backend and a React + Vite frontend. It supports user registration, login, credit risk scoring, and personalized recommendations based on application input.

## Key Features

- User authentication with secure password hashing
- Credit risk prediction endpoint
- Risk score and credit score calculation
- Recommendation engine for financial guidance
- Modern React frontend with protected routes
- Structured backend using SQLAlchemy and FastAPI

## Repository Structure

- `backend/` - FastAPI application, database models, authentication, prediction endpoints, and CRUD operations
- `frontend/` - React + Vite user interface for registration, login, and prediction workflows
- `data/` - raw and processed datasets used for model training and analysis
- `ml/` - preprocessing, feature engineering, and training scripts
- `docs/` - project documentation, analysis notes, and model reports
- `reports/` - evaluation and explainability reports

## Requirements

- Python 3.11+ (recommended)
- Node.js 18+ / npm or Yarn
- PostgreSQL or another database supported by SQLAlchemy, or SQLite for local testing

## Backend Setup

1. Open a terminal in the project root:

```bash
cd d:\Creditwise-Ai\creditwise_ai
```

2. Create and activate a Python virtual environment:

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

3. Install backend dependencies:

```powershell
pip install -r requirements.txt
```

4. Create a `.env` file in the `backend/` directory with the following variables:

```env
DATABASE_URL=sqlite:///./creditwise.db
SECRET_KEY=replace-with-a-secure-random-string
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=60
```

5. Start the backend API:

```powershell
python -m uvicorn backend.main:app --reload
```

The API will be available at `http://127.0.0.1:8000`.

## Frontend Setup

1. Change into the frontend directory:

```powershell
cd frontend
```

2. Install dependencies:

```powershell
npm install
```

3. Start the development server:

```powershell
npm run dev
```

The frontend will typically run at `http://localhost:5173`.

## Usage

### Register a new user

POST `/register`

Request body:

```json
{
  "full_name": "Jane Doe",
  "email": "jane@example.com",
  "password": "securePassword123"
}
```

### Login

POST `/login`

Request body:

```json
{
  "email": "jane@example.com",
  "password": "securePassword123"
}
```

### Predict credit risk

POST `/predict`

Request body example:

```json
{
  "age": 35,
  "annual_income": 80000,
  "loan_amount": 12000,
  "annuity_amount": 250,
  "employment_years": 6,
  "number_of_children": 1,
  "credit_inquiries": 2,
  "ext_score_mean": 0.65
}
```

## Notes

- The backend uses SQLAlchemy ORM models and a database URL from `.env`.
- Authentication relies on JWT tokens generated by `backend/app/security.py`.
- The frontend connects to the backend through `frontend/src/services/api.js` and `frontend/src/services/authService.js`.
